# Kimi Web Assistant — 需求报告

## 1. 项目背景与目标

**Kimi 网页端**目前存在以下体验痛点：
- 长对话缺乏**轮次进度条**，无法快速定位与跳转。
- **置顶功能仅支持单个对话**，无法对多个重要对话进行收藏与分类整理。
- 对话中的 **LaTeX 公式** 没有一键复制入口，学术与编程场景下效率低。
- 缺少**跨对话搜索**能力，历史内容难以快速回溯。

本项目目标：通过**浏览器插件（Browser Extension / 浏览器扩展）**的形式，在不依赖官方接口改造的前提下，为 Kimi 网页端注入增强能力，解决上述痛点。

---

## 2. 为什么选用浏览器插件

| 方案 | 可行性 | 说明 |
|------|--------|------|
| **浏览器插件** | ⭐⭐⭐ 高 | 直接注入 Content Script，读取 DOM、追加 UI、持久化本地数据，用户可独立安装。 |
| 用户脚本（Tampermonkey） | ⭐⭐ 中 | 同样可注入页面，但代码维护、权限管理、UI 组件化不如扩展规范。 |
| 独立桌面端/爬虫 | ⭐ 低 | 需要逆向 API、维护成本高、存在账号与合规风险。 |

**结论**：浏览器插件（Manifest V3）是最平衡、最可持续的实现方式。

---

## 3. 用户画像与核心场景

### 用户画像
- **高频学术用户**：经常与 Kimi 进行多轮数学/物理推导，需要复制 LaTeX。
- **多线程工作者**：同时维护多个对话主题（编程、写作、调研），希望收藏并快速切换。
- **长文阅读/写作用户**：单条对话极长，需要类似“目录/进度条”的导航能力。

### 核心场景
1. **场景 A — 长对话导航**：用户打开一个已有 50+ 轮次的对话，通过右侧悬浮进度条快速滚动到第 30 轮。
2. **场景 B — 公式复制**：用户在回答中看到渲染好的数学公式，点击按钮直接复制原始 LaTeX 代码到剪贴板。
3. **场景 C — 多对话收藏**：用户在侧边栏对多个对话点击“收藏”，并归类到自定义集合（如“论文精读”、“项目需求”）。
4. **场景 D — 跨对话搜索**：用户在插件 Popup 中输入关键词，快速定位到已收藏对话中的相关轮次。

## 4. 功能需求

### 4.1 对话轮次进度条（Conversation Navigator）
- **FN-01** 在对话页面右侧注入一个可折叠的迷你进度条/导航面板。
- **FN-02** 进度条按轮次（用户提问 / AI 回复）分段，支持视觉区分（用户/模型、长度比例）。
- **FN-03** 点击进度条上的某一段，页面自动平滑滚动到对应轮次。
- **FN-04** 当前视口所在轮次在进度条中高亮显示。
- **FN-05** 支持缩略预览：鼠标悬停时显示该轮次的开头几行文字摘要。

### 4.2 LaTeX 快捷复制（LaTeX Copier）
- **FN-06** 自动识别对话中渲染后的数学公式块（通常对应 `<math>` 或特定 class 的 DOM 节点）。
- **FN-07** 在公式右上角添加一个“复制 LaTeX”按钮，点击后将原始 LaTeX 源码写入剪贴板。
- **FN-08** 复制成功后给予视觉反馈（如按钮图标变化或 Toast 提示）。

### 4.3 多对话收藏与整理（Collection Manager）
- **FN-09** 在侧边栏每个对话项旁注入“收藏”按钮（星标/心形）。
- **FN-10** 收藏的对话数据（标题、链接、时间戳）持久化保存到浏览器存储中。
- **FN-11** 支持创建自定义集合（Collection），一个对话可归属多个集合。
- **FN-12** 支持取消收藏、从集合移除、重命名集合。
- **FN-13** 在插件 Popup 中展示收藏列表与集合树，点击直接打开对应对话。

### 4.4 跨对话快速搜索（Quick Search）
- **FN-14** 在插件 Popup 中提供搜索框，支持按标题和摘要搜索已收藏对话。
- **FN-15**（进阶）支持在已缓存的对话内容中搜索关键词，返回匹配的轮次与链接。

### 4.5 设置与数据管理
- **FN-16** 提供设置页，可开关各功能模块（进度条、LaTeX 复制、收藏按钮）。
- **FN-17** 支持导出/导入收藏数据（JSON 格式），便于备份与迁移。

---

## 5. 非功能需求

- **NF-01 兼容性**：支持 Chrome / Edge / Firefox（尽量使用跨浏览器兼容的 WebExtensions API）。
- **NF-02 性能**：Content Script 注入不应明显拖慢页面加载；DOM 监听使用节流/防抖。
- **NF-03 隐私**：所有数据默认保存在本地 `chrome.storage.local`，不上传第三方服务器。
- **NF-04 可维护性**：采用 Manifest V3，代码模块化，便于后续跟随 Kimi 前端改版调整选择器。

## 6. 技术架构草案

```
kimi-web-assistant/
├── manifest.json              # Manifest V3 配置
├── src/
│   ├── content/
│   │   ├── navigator.js       # 进度条注入与滚动定位
│   │   ├── latex-copier.js    # LaTeX 按钮注入与复制
│   │   ├── collection-inject.js # 侧边栏收藏按钮注入
│   │   └── observer.js        # DOM 变化观察与调度
│   ├── background/
│   │   └── service-worker.js  # 跨标签通信、存储代理
│   ├── popup/
│   │   ├── popup.html
│   │   ├── popup.css
│   │   └── popup.js           # 收藏列表、搜索、集合管理
│   ├── shared/
│   │   ├── storage.js         # chrome.storage 封装
│   │   ├── selectors.js       # Kimi DOM 选择器常量
│   │   └── utils.js           # 通用工具函数
│   └── assets/
│       └── icons/             # 插件图标
└── docs/
    └── REQUIREMENTS.md        # 本文件
```

## 7. 里程碑规划

| 阶段 | 目标 | 核心交付 |

## 8. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Kimi 前端 DOM 结构变更 | 高 | 将选择器集中管理，缩短修复周期；关注官方更新日志。 |
| 浏览器安全策略收紧 | 中 | 遵循 Manifest V3 最佳实践，避免使用已废弃 API。 |
| 数学公式渲染方式改变 | 中 | 同时支持 MathJax / KaTeX / 原生 math 标签多种解析策略。 |
